01 ⚡ Hardware & Arquitetura BASE ▶
CPU & Núcleos
Como o processador executa instruções. Clock, pipeline, múltiplos núcleos.
Assembly
Linguagem mais próxima do hardware. Tudo que você escreve vira isso.
Registradores
Memória ultra-rápida dentro da CPU. EAX, EBX, ESP...
Ciclo Fetch-Decode-Execute
O loop eterno da CPU: busca instrução → decodifica → executa.
Cache L1/L2/L3
Memórias intermediárias entre CPU e RAM. Cada nível mais lento e maior.
Von Neumann
Arquitetura base de todo computador moderno. Programa na memória.
por que isso importa Quando você entende que um loop em Python vira dezenas de instruções Assembly que a CPU executa uma por vez por núcleo, você começa a entender performance, bottlenecks e por que certas escolhas de código custam mais caro.
02 🧠 Sistema Operacional & Memória CORE ▶
Threads
Unidade de execução dentro de um processo. Um processo pode ter várias threads.
Processos
Programa em execução. Tem memória própria isolada.
Stack vs Heap
Stack: variáveis locais, rápida, limitada. Heap: alocação dinâmica, flexível.
GIL do Python
Global Interpreter Lock. Por que Python não usa múltiplos núcleos facilmente.
Scheduler
O SO decide qual thread roda agora. Context switch tem custo.
Syscalls
Seu programa pedindo algo ao SO: ler arquivo, abrir socket, alocar memória.
por que isso importa Seu chefe pediu explicitamente threads e núcleos. Django em produção roda múltiplos workers. Entender threads te explica deadlocks, race conditions e por que async/await existe.
03 🔤 C e como Python funciona por baixo LINGUAGEM ▶
CPython
O interpretador Python escrito em C. Seu código Python vira bytecode que C executa.
Bytecode
Código intermediário. Python compila pra .pyc antes de executar.
Como um for funciona
Chama __iter__, depois __next__ em loop até StopIteration. Tudo em C.
Ponteiros (C)
Variável que guarda endereço de memória. Base de como Python gerencia objetos.
Garbage Collector
Python libera memória automaticamente via reference counting + GC cíclico.
dis module
dis.dis(funcao) mostra o bytecode Python. Você vê o que realmente executa.
por que isso importa Quando você usa uma lista vs um gerador, um dict vs uma lista, você toma decisões de memória e performance. Entender o CPython te faz escolher a estrutura certa, não a que parece mais fácil.
04 🌐 Redes, HTTP e Pacotes WEB ▶
Pacotes TCP/IP
Como dados viajam na rede. Fragmentados em pacotes com header e payload.
HTTP/HTTPS
Protocolo de aplicação. Request-Response. Verbos: GET, POST, PUT, DELETE.
DNS
Traduz nome de domínio em IP. Antes de qualquer conexão isso acontece.
Sockets
API do SO pra comunicação de rede. Django usa isso por baixo.
Headers HTTP
Metadados da requisição: Content-Type, Authorization, Cache-Control...
Status Codes
200, 201, 400, 401, 403, 404, 500. Cada um tem significado preciso.
por que isso importa Seu chefe citou pacotes HTTP e aplicação. Todo sistema web é HTTP. Entender o protocolo te permite debugar problemas de rede, implementar autenticação correta e usar cache do jeito certo.
05 🗄️ Banco de Dados, ORM e Cache PRODUÇÃO ▶
SQL puro
SELECT, JOIN, INDEX, EXPLAIN. Base antes de qualquer ORM.
Django ORM
Traduz Python em SQL. Model.objects.filter() vira um SELECT com WHERE.
N+1 Problem
Loop que faz uma query por item. select_related() e prefetch_related() resolvem.
Cache (Redis)
Memória ultra-rápida. Evita repetir queries pesadas. Seu chefe citou isso.
Índices
Estrutura que acelera busca. Sem índice = full table scan = lento.
Transactions
ACID: Atomicidade, Consistência, Isolamento, Durabilidade. Dados íntegros.
por que isso importa Seu chefe citou ORM, cache e "como fazer sem gastar muitas requisições". O N+1 problem é o erro mais comum de devs júnior. Quem entende SQL não usa ORM às cegas — vê o que está sendo gerado e otimiza.

// termos que seu chefe usou — definições precisas
ORM
Object-Relational Mapping. Camada que traduz objetos Python em SQL automaticamente. No Django: User.objects.filter(ativo=True) vira SELECT * FROM users WHERE ativo=1. O risco é usar sem saber o SQL que está sendo gerado.
seu chefe citou
Cache
Armazenamento temporário de dados frequentemente acessados. Redis é o mais comum com Django. Resultado de uma query pesada vai pro cache, próximas requisições não batem no banco. Seu chefe falou em "gastar menos requisições".
seu chefe citou
Thread
Unidade de execução dentro de um processo. Um processo pode ter várias threads rodando "ao mesmo tempo" (compartilhando a mesma memória). Python tem o GIL que limita threads a rodar uma por vez no CPython.
seu chefe citou
Assembly
Linguagem mais próxima das instruções do processador. MOV, ADD, JMP são instruções reais da CPU. C compila pra Assembly. Python (via CPython em C) vira Assembly no final. Tudo vira Assembly.
seu chefe citou
Pacote HTTP
Unidade de dados que trafega na rede. Tem header (metadados: método, URL, tipo de conteúdo, autenticação) e body (dados em si: JSON, HTML, arquivo). Django recebe e responde com pacotes HTTP.
seu chefe citou
CPython
A implementação padrão do Python, escrita em C. Quando você instala Python do python.org, está instalando CPython. Existe também PyPy (escrito em Python), Jython (Java) e outros.
N+1 Problem
Bug de performance comum com ORM. Você busca N registros, depois faz 1 query pra cada = N+1 queries. Solução Django: select_related() para FK e prefetch_related() para M2M.
GIL
Global Interpreter Lock. Mecanismo do CPython que garante que apenas uma thread execute bytecode Python por vez. Por isso Python não aproveita múltiplos núcleos facilmente com threads. Solução: multiprocessing ou async.
